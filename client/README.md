# HireMe AI 채용 관리 시스템

HireMe는 AI 기반 채용 관리 시스템으로, 구직자와 채용 담당자를 위한 종합적인 플랫폼입니다.

## 🚀 주요 기능

### 📋 채용 공고 관리
- **공고 목록**: 다양한 채용 공고를 카테고리별로 확인
- **상세 페이지**: 각 공고의 자세한 정보와 지원 기능
- **지원 통계**: 지원자 수, 조회수, 마감일 등 실시간 정보

### 📝 지원서 관리
- **입력 페이지**: 온라인 지원서 작성
- **PDF 등록**: 최대 3개까지 PDF 파일 업로드
- **지원 현황**: 지원한 공고들의 상태 확인

### 💼 포트폴리오 관리
- **GitHub 연동**: GitHub 저장소 경로 등록
- **프로젝트 관리**: 주요 프로젝트 및 기술 스택 표시
- **포트폴리오 통계**: 활동 현황 및 기여도 분석

### 🤖 AI 인재 추천
- **스마트 매칭**: AI 기반 인재 추천 시스템
- **필터링**: 경력, 기술, 지역별 필터링
- **추천 통계**: 매칭률 및 추천 현황

### 🎯 면접 관리
- **면접 방식 선택**: Zoom, 영상 업로드, 대면 면접
- **면접 일정**: 면접 일정 관리 및 알림
- **면접 준비**: 면접 가이드 및 팁 제공

## 🛠 기술 스택

### Frontend
- **React 18**: 최신 React 기능 활용
- **TypeScript**: 타입 안정성 보장
- **Material-UI v7**: 모던한 UI 컴포넌트
- **Font Awesome**: 일관된 아이콘 시스템
- **React Router**: SPA 라우팅

### 개발 도구
- **ESLint**: 코드 품질 관리
- **TypeScript**: 정적 타입 검사
- **Webpack**: 모듈 번들링

## 🎨 UI/UX 특징

### 반응형 디자인
- **모바일 우선**: 모든 디바이스에서 최적화된 경험
- **CSS Grid**: 유연한 레이아웃 시스템
- **Material Design**: 일관된 디자인 언어

### 사용자 경험
- **직관적 네비게이션**: 5개 메인 메뉴로 구성
- **상세 정보 제공**: 각 페이지별 충분한 정보 제공
- **상태 피드백**: 로딩, 성공, 오류 상태 표시

## 🚀 시작하기

### 필수 요구사항
- Node.js 16.0 이상
- npm 또는 yarn

### 설치 및 실행

```bash
# 의존성 설치
npm install

# 개발 서버 실행
npm start

# 빌드
npm run build
```

### 환경 설정
- **포트**: 기본 3000번 포트 사용
- **브라우저**: Chrome, Firefox, Safari 지원

## 📋 주요 페이지 설명

### 1. 홈페이지 (`/`)
- 시스템 소개 및 주요 기능 안내
- 통계 정보 및 CTA 섹션

### 2. 채용 공고 (`/jobs`)
- 채용 공고 목록 표시
- 카테고리별 필터링
- "지원하기" 버튼으로 상세페이지 이동

### 3. 채용 상세 (`/jobs/:id`)
- 공고별 상세 정보
- 자격 요건, 우대 사항, 복리후생
- 지원자 통계 및 지원하기 기능

### 4. 지원서 관리 (`/applications`)
- **입력 페이지**: 온라인 지원서 작성
- **PDF 등록**: 파일 업로드 기능
- 탭 기반 인터페이스

### 5. 포트폴리오 (`/portfolio`)
- GitHub 저장소 연동
- 프로젝트 관리 및 통계
- 기술 스택 시각화

### 6. 인재 추천 (`/recommendations`)
- AI 기반 인재 추천
- 필터링 및 정렬 기능
- 추천 통계 정보

### 7. 면접 관리 (`/interviews`)
- 면접 방식 선택 (Zoom/영상/대면)
- 면접 일정 관리
- 면접 준비 가이드

## 🔧 개발 가이드

### 코드 스타일
- **TypeScript**: 엄격한 타입 체크
- **ESLint**: 코드 품질 규칙 적용

- **함수형 컴포넌트**: React Hooks 활용

### 상태 관리
- **React Hooks**: useState, useEffect 활용
- **라우터 상태**: useParams, useNavigate 활용
- **로컬 상태**: 컴포넌트별 상태 관리
